# Реализация http-сервера в виде сервлета

## Задача

Вместо запуска сервера в виде слушателя сокета - создаем сервлет, 
который запускается внутри контейнера сервлетов.

## Состав компонентов

src\main\webapp\WEB-INF\web.xml - описание сервлета, чтобы контейнер знал его состав

PostController.java - первый слой обработки запроса - контроллер. Содержит хендлеры, которые запускаются в зависимости от параметров запроса.
Хендлеры занимаются только разбором/форматированием запроса и отправкой на дальнейшую обработку. Все функционально полезные действия с объектами делаются в других слоях.

MainServlet.java - сервлет. Для работы переопределяет функции init и service: init запускается при старте сервлета, 
в нем инициализируются все необходимые для работы сервлета классы, service - получает на вход запрос, 
решает в какой метод контроллера отправить запрос и отправляет.

PostService.java - прикладной слой, к которому обращается контроллер после первичной обработки запроса.
(спойлер: также в этом слое делаются обработчики сообщений из брокеров обмена).

PostRepository.java - слой управления хранением данных (в случае необходимости долговременного хранения, как на постоянных носителях, так и in-memory).

Post.java - модель данных, т.е. описание основной сущности бизнес-уровня, которой/которыми оперируем через запросы
(спойлер: также бывают вспомогательные сущности, которые не считаются моделями, а могут быть например DTO - data transfer object - объекты для оперириования частичными данными из которых в процессе работы создаются модели)

NotFoundException.java - класс кастомного исключения.

## Ветки

feature/di-annotation - демонстрация использования аннотаций, чтобы не выполнять инициализацию классов контроллера, сервиса и репозитория вручную в коде

feature/di-java - демонстрация конфигурации через репозиторий бинов
